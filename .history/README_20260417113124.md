# AI E-Book Generator

An end-to-end AI-assisted document and eBook authoring platform built with Next.js, TypeScript, Supabase, and Tiptap.

Users can authenticate, create documents from templates, write with a rich page-based editor, generate content with AI, and export files in multiple formats.

## Key Features

- Secure authentication flow using Supabase Auth (sign-up, login, protected dashboard routes).
- Personalized dashboard with recent documents, quick actions, and user stats.
- Template-driven document creation with support for public and user-owned templates.
- Rich text editing with Tiptap, including headings, lists, tables, formatting, code blocks, and images.
- Page-centric writing experience with configurable headers, footers, borders, and page numbers.
- eBook metadata capture workflow for author/title/publisher setup.
- AI-powered content generation endpoint using Groq models with context-aware continuation and expansion.
- Auto-save and manual save support with word count tracking.
- Multi-format export support: Markdown, HTML, print-ready HTML (PDF workflow), and DOCX.
- Row Level Security (RLS) enabled for user data isolation in Supabase.

## Functional Overview

This section describes what the platform does from a user workflow point of view.

### 1. Authentication and Onboarding

- A new user signs up from `/auth/sign-up`.
- After registration and confirmation, the user is redirected to the dashboard.
- Returning users sign in from `/auth/login`.
- Protected routes such as `/dashboard` and `/editor/*` require authentication.

### 2. Dashboard and Document Management

- The dashboard shows recent documents, quick actions, and high-level usage stats.
- Users can browse all their documents in `/dashboard/documents`.
- Users can update personal details from `/dashboard/profile` and app preferences from `/dashboard/settings`.

### 3. Template-Based Creation

- Users can browse templates in `/dashboard/templates`.
- A new document can be started from a template or from a blank editor.
- Templates support both public starter content and user-owned content patterns.

### 4. Editor Workflow

- Users create or open a document in `/editor/new` or `/editor/[id]`.
- The editor supports rich formatting: headings, lists, tables, code blocks, images, and structural layout.
- Page settings allow control of borders, headers, footers, and page numbering.
- Metadata (title, author, publisher, etc.) can be configured for eBook output.

### 5. AI-Assisted Writing

- Users send prompts through the AI assistant to generate or continue content.
- Generation can use document context for more relevant output.
- AI output can be inserted into the editor and refined with manual edits.

### 6. Saving and Export

- Documents support save workflows with content persistence in Supabase.
- Export is available through `/api/export` in multiple formats.
- Supported outputs: `markdown`, `html`, `pdf` (print HTML flow), and `docx`.

### 7. Security and Data Isolation

- Supabase Auth identifies users.
- Supabase RLS policies enforce per-user data access in core tables.
- API routes for AI and export are designed for authenticated usage.

## Main User Flows (At a Glance)

1. Sign up/sign in.
2. Open dashboard and choose a template (or start blank).
3. Write in the editor and use AI to draft/expand content.
4. Adjust page settings and metadata.
5. Save and export to the desired format.

## Tech Stack

- Frontend: Next.js 16 (App Router), React 19, TypeScript
- Styling/UI: Tailwind CSS, Radix UI, custom UI components
- Editor: Tiptap + Tiptap Pro extensions (Pages, AI)
- Backend: Next.js Route Handlers
- Database/Auth: Supabase (PostgreSQL + Auth + RLS)
- AI Provider: Groq Chat Completions API
- Export Tooling: docx, html-docx-js, jspdf, mammoth, puppeteer (project dependencies)

## Prerequisites

- Node.js 20+
- pnpm 9+ (recommended)
- Supabase project
- Groq API key
- (Optional, for Tiptap AI extension) Tiptap Cloud app credentials

## Environment Variables

Create a `.env.local` file in the project root:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key

# Used for email redirect during sign-up flow in development
NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL=http://localhost:3000/dashboard

GROQ_API_KEY=your_groq_api_key

# Optional but recommended for Tiptap AI extension
NEXT_PUBLIC_TIPTAP_APP_ID=your_tiptap_app_id
NEXT_PUBLIC_TIPTAP_API_TOKEN=your_tiptap_api_token
```

## Local Setup

1. Install dependencies:

```bash
pnpm install
```

2. Configure environment variables in `.env.local`.

3. Initialize Supabase schema:

- Run `scripts/001_create_schema.sql` in your Supabase SQL editor.
- Run `scripts/002_seed_templates.sql` to seed starter templates.

4. Start development server:

```bash
pnpm dev
```

5. Open the app:

- http://localhost:3000

## API Endpoints

- `POST /api/ai/generate`
	- Authenticated route for generating AI content.
	- Supports document context and document type aware prompts.

- `POST /api/export`
	- Authenticated route for exporting user documents.
	- Supported formats: `markdown`, `html`, `pdf` (print HTML), `docx`.

## Data Model (High-Level)

- `profiles` - User profile information
- `documents` - User-authored content with metadata and status
- `templates` - Public and user-created templates
- `ai_generations` - Prompt, model, token usage, and generated output logs

All core tables are protected with RLS policies scoped to authenticated users.

## Deployment
